# Scoring Rules for Probabilistic Forecasting

## Project Overview

Seminar talk, written report and companion code on **proper scoring rules**, the theory of how to evaluate forecasts that come as full probability distributions rather than point predictions.

The material was prepared for the *Uncertainty in Machine Learning* seminar (TU Berlin, talk given 27 November 2025) and follows Gneiting & Raftery's classic paper [*Strictly Proper Scoring Rules, Prediction, and Estimation*](https://doi.org/10.1198/016214506000001437) (JASA, 2007).

- Slides: [Seminar_Talk/main.tex](Seminar_Talk/main.tex)
- Written report: [Seminar_Talk/Summary/Summary.tex](Seminar_Talk/Summary/Summary.tex)
- Figure code: [code/](code/)

## Motivation

A weather service that says *"23 °C tomorrow"* is easy to grade; one that hands
you a whole distribution over temperatures is not. A **scoring rule**
$S(P, \omega)$ rewards a forecast distribution $P$ once the outcome $\omega$ is
observed. The crucial property is **(strict) properness**: honesty must be the
optimal strategy. In the reward convention used throughout this project
(higher scores are better), a scoring rule is proper if

$$S(Q, Q) \geq S(P, Q) \quad \text{for all } P, Q,$$

and strictly proper if $S(Q, Q) > S(P, Q)$ for all $P \neq Q$. A forecaster
who believes the truth is $Q$ then maximizes their expected score by actually
reporting $Q$ — a strictly proper rule leaves no room to game the metric by
hedging or exaggerating confidence. The associated divergence
$d(P, Q) = S(Q, Q) - S(P, Q) \geq 0$ measures how much a dishonest or wrong
forecast loses in expectation.

## Topics Covered

- Properness and strict properness of scoring rules
- Convex entropy functions and **Bregman divergences**
- Categorical scores: **Brier**, **logarithmic**, **zero-one**
- Scores for densities (quadratic, pseudospherical, logarithmic) and their pitfalls (point-mass exploits)
- The **Continuous Ranked Probability Score (CRPS)** for CDFs
- **Kernel scores** as a unifying construction (CRPS and Brier as special cases)
- Proper scores for **quantile** forecasts, interval score, Value at Risk
- Using scoring rules to compare forecasting models

### The theory in pictures

All four figures are generated by [code/visualize_theory.py](code/visualize_theory.py).

**Properness: honesty is the best policy.** For a binary event with true
probability $q = 0.7$, the *expected* Brier and logarithmic scores are
uniquely maximized by reporting $p = q$. The zero-one score is proper but
**not strictly** proper, every report on the correct side of $1/2$ scores the
same, so it cannot distinguish an honest forecaster from a lazy one.

![Properness](code/figures/properness.png)

**Divergences: the geometry behind proper scores.** Every proper score has a
convex entropy $G$, and its divergence $d(P, Q) = S(Q,Q) - S(P,Q)$ is the gap
between $G$ and its tangent at the forecast, a **Bregman divergence**. For the
logarithmic score this gap is exactly the Kullback–Leibler divergence.

![Bregman divergence](code/figures/bregman_divergence.png)

**CRPS: scoring cumulative distribution functions.** The CRPS is the
(negative) squared area between the forecast CDF and the step function of the
observation. Unlike the log score it is sensitive to *distance*: a near miss
beats a far miss, and sharpness pays off near the center but costs in the
tails.

![CRPS](code/figures/crps.png)

**Monte Carlo: proper scores punish dishonest uncertainty.** Nature draws
from $\mathcal{N}(0,1)$; forecasters report $\mathcal{N}(0,\sigma^2)$ with
varying sharpness. Averaged over 50 000 draws, both the log score and the
CRPS peak at the honest $\sigma = 1$, neither overconfidence nor
underconfidence helps.

![Honesty experiment](code/figures/honesty_experiment.png)

## Installation

With pip:

```bash
pip install -r requirements.txt
```

Or with conda:

```bash
conda env create -f environment.yml
conda activate scoring-rules
```

## Reproducing the Figures

```bash
python code/generate_all_figures.py   # everything in one go
# or equivalently
make figures
```

Individual scripts can also be run directly, e.g.
`python code/visualize_theory.py`. All output lands in
[code/figures/](code/figures/). The Monte Carlo experiment uses a fixed random
seed, so the figures are fully reproducible. The talk scripts use real LaTeX
for text rendering when a `latex` binary is on the `PATH` and fall back to
matplotlib's mathtext otherwise.

To run the test suite:

```bash
python -m pytest    # or: make test
```

## Compiling the Slides

Requires a LaTeX distribution with `latexmk`:

make slides

cd Seminar_Talk && latexmk -pdf main.tex
